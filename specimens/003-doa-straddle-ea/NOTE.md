# 検体003 — 検体カード

| 項目 | 内容 |
|------|------|
| 検体番号 | 003 |
| 型 | ブレイクアウト・ストラドル型EA(無条件両建て / 待機注文追従型) |
| 想定設計意図 | レンジ収束時にBuyStop/SellStopを同時設置し、ボラティリティ拡張時のブレイクアウトを狙う(コードから逆算) |
| 稼働実態 | 環境認識フィルタが皆無であり、論理破綻した数式で極小TP/SLの待機注文を恒常的に発注し続ける |
| 総行数 | 抜粋による診断のため不問 |
| 診断者 | Semura Lab. |

## 診断サマリ

- **FATAL-A**: 追随ロジックの数式破綻。`OrderOpenPrice()+Bid`という価格の加算処理が行われており、距離判定として論理的に成立しない。
- **FATAL-B**: エントリーフィルタの完全不在。相場分析を一切行わず、無条件で両建ての待機注文を置き続ける。
- **FATAL-C**: スプレッド摩擦への無防備。TP=2.0pipsという極小レンジに対し、平均スプレッドが実質的な利益の大部分を削る。
- **D(補足)**: 両建て構造による二重コスト。一方が約定する構造上、もう一方は高確率で損切りに巻き込まれるか不利な位置へ引きずられる。
- **E**: 飾りパラメータ。`Periods=14`等、インジケーター用の変数が宣言のみで未使用(デッドコード)。

## CAP判定

- 未来リーク: 白(判断ロジック自体が不在のため該当せず)
- Close-cap: 白
- プラセボ: 黒(未使用のインジケーター用パラメータが残存)
- 表示と実装の不一致: 黒(スリッページ制御の宣言があるが実働関数への引数は0)
- カーブフィット: 灰(最適化するパラメータ自体が機能していないため判定不能)

## 数理

名目上のTP=2.0pips / SL=1.8pipsという極小レンジにおいて、平均スプレッド0.6pipsを考慮した実効損益は「利益1.4pips / 損失2.4pips」となる。この実効値における損益分岐勝率は約63.1%である。
本検体には相場の優位性を判定する方向エッジが存在しないため、勝率がこの分岐点を超える数理的根拠はない。純粋なスプレッド摩擦と両建ての構造的損失が蓄積し、期待値はマイナスへ収束する。
