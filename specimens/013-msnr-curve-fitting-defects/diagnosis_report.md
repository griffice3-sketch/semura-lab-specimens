## 0. 前提解析 — ロジックの目的・狙い・思想

コードから読み取れる設計思想は以下の通り。

- **本体思想:** HTF（W1/D1/H4/H1）のSNR（Body/GAP）を骨格とし、M5で「10レイヤー（L1〜L10：Touch / Sweep / MISS / Engulf / GAP / TL / QML / QML-Engulf / BO-Retest / CRT）」を評価。同一ゾーンに重なった信号を**コンフルエンス・クラスター**として束ね、1クラスタ1エントリーを建前とする。
- **利確思想:** DOL（Draw On Liquidity＝PDH/PWH/スイング流動性）を第一目標とし、なければSTD/RR投影で **最低5R** を要求。4Rで20%部分利確しBE移動。
- **車線（レーン）思想:** REACTION（反応）／MOMENTUM（順張り）／STRONG TREND（暴走ブレイク）／TREND-ARMED PULLBACK（押し目）の**4レーン併走**。
- **運用思想:** Asia/EU/USセッション制御、equity curveに応じたリスク可変（SAFE/GROWTH/LOCKDOWN）、連敗・DDサーキットブレーカ。

**狙いの本質:** 「ゴールドの暴走相場をSNR反応＋順張りの両面で拾い、5R高RRで非対称ペイオフを得る」低勝率・高ペイオフ型。

しかし——その思想を実装する過程で、**致命的な病巣**が混入している。以下、冷徹に解剖する。

## 1. 総合所見（生存能力の有無）

### 判定: **生存能力なし（現状）**

理由は3点に集約される。

1. **検体が「バックテスト結果に対して後付けで成形された塊」である。** バージョン履歴 v5.11→v5.31 の全域に、`layerMask == 217` `== 541` `== 805` `== 301` `== 705` `== 193` といった**ハードコードされたマジックナンバー群**が散在し、コメントには明示的に *"worst groups from longer test"* *"good momentum groups from v5.25"* *"L1+L10 was the biggest loser"* と記されている。これは**過去データに勝ったものを残し、負けたものを名指しで除外した結果**であり、カーブフィッティングの教科書的標本だ。汎化性能はゼロに近い。

2. **「単一ポジション」設計を真っ向から破っている。** `NoTotalTradeLimit=true` / `MaxTotalPositions=0`（無制限）に加え、4レーンが同一tick内で順次エントリーを試みる。1バーで複数建玉が成立しうる構造であり、規格違反。

3. **検証不能。** 月利・PF・DD・WFA結果が一切添付されていない。コードは「5Rを取れれば勝つ」と主張するだけで、**期待値が正であることを示す証拠が皆無**。診断医は数値の裏付けなき主張を採用しない。

## 3. 発見された致命的脆弱性

### 🔴 病巣A：カーブフィッティングの全身転移（最重症）
レーン・コンボ・ステートごとに「過去に勝った組み合わせだけを通す」フィルタが層状に積まれている（`WhitelistTag520`, `MomentumLayerAllowed527`, `BlockedWorstNormalBuy520`, `DisableNormalMarket`, `DisableReactionRangeLock`…）。これは**ロジックではなく、過去への暗記**。未知の13年に対する優位性を一切保証しない。**v5.31という版番号そのものが、30回以上のデータ覗き見（data snooping）の証拠**である。

### 🔴 病巣B：多重保有による「隠れたリスク増幅」
1クラスタ1建玉という建前に反し、REACTION＋MOMENTUM＋STRONG TREND＋PULLBACKが独立に発火する。相関の高い同方向ポジションが同時保有されれば、**1イベントでの実効リスクは設計値の数倍**に膨らむ。連敗ガードの`MaxBalanceDDPausePct=28%`という許容DDの大きさが、この危険性を裏付けている。

### 🟠 病巣C：リスク設定の二重基準（リアル乖離）
`RiskPercentPerOrder=1%`でロット計算後、`NormalizeVolume531`が`MaxLotPerTrade=0.01`で問答無用にキャップ。**口座が成長してもロットは0.01に固定**され、リスク%が形骸化。バックテストの月利10%主張とリアル挙動が構造的に一致しない。

### 🟠 病巣D：スプレッド/コスト摩擦の軽視
0.6pips摩擦の期待利得反映なし、拡大時間帯回避なし、`MaxSpread=350pt`は金にしては緩い。**5R設計は一見ロバストだが、エントリー精度がスプレッド次第で容易に崩れる**M5スキャル的タッチ判定（`TouchBufferPrice=0.25`）に依存しており、摩擦感度が高い。

### 🟡 病巣E：複雑性そのものが脆弱性
入力パラメータ数百、レーン4本、レイヤー10、マスク定数多数。**自由度が高すぎてWFAでの安定パラメータ域が存在しない**可能性が極めて高い。「綺麗な検体（CleanStudy）」を名乗るが、内部は最適化の瘢痕組織で覆われている。

## 4. Semura Lab規格への再生処方箋

延命ではなく**再構築**を前提とした処方を示す。

### 処方1：マジックナンバーの全摘出（最優先）
`layerMask == 数値` で過去結果を焼き込んだ全分岐（v5.20/v5.25/v5.27/v5.29由来）を**完全削除**。残すのは「論理的に説明可能な条件」のみ。レイヤーは**最大3つ**まで削ぎ落とし、各々の優位性を単体でフォワード検証せよ。

### 処方2：単一ポジション化の強制
`NoTotalTradeLimit=false` / `MaxTotalPositions=1` を**ハードロック**。4レーンは廃止し、**1つの整合した発火ロジックに統合**。同方向多重保有を物理的に不可能にせよ。

### 処方3：リスク基準の一本化
`MaxLotPerTrade`キャップを撤廃し、リスク%を真の基準に。バックテストとリアルで同一挙動を保証すること。これが成らない限り月利は計測不能。

### 処方4：スプレッド摩擦の内製化
- ロールオーバー帯（例：サーバー23:55〜00:10）と主要指標時刻のエントリー禁止フィルタを実装。
- 全エントリーに**0.6pips固定摩擦＋手数料**を織り込んだ純期待値を算出し、`MinRR`をこのコスト後で再定義。

### 処方5：WFA前提の証拠提出
削ぎ落とした素体で **13年アンカードWFA（IS:OS=3:1等）** を実施し、PF・月利・最大DD・各窓の一貫性を提出せよ。マジックナンバーを抜いた素体が**それでも正の期待値を示すか**——それが唯一の生存試験である。
