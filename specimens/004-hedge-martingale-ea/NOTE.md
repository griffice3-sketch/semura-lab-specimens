# 検体004 — 検体カード

| 項目 | 内容 |
|------|------|
| 検体番号 | 004 |
| 型 | 両建てヘッジ・変則マーチンゲール型EA(両建てロック / 逆行ナンピン加重平均決済) |
| 想定設計意図 | 常時Buy/Sellを保有し、逆行時にマーチンゲールでナンピン。建値平均+微益で全決済する「含み損を隠蔽しつつ高勝率を演出する」構造(コードから逆算) |
| 稼働実態 | 環境認識フィルタが皆無であり、不規則な配列ロットを無制限に積み上げ、最終的に配列エラー(クラッシュ)か証拠金枯渇で確実に破綻する |
| 総行数 | 抜粋による診断のため不問 |
| 診断者 | Semura Lab. |

## 診断サマリ

- **FATAL-A**: 口座破綻を約束するナンピン構造。最大15段目（初期の185倍）までロットが膨張する、カーブフィットされた狂気的な不規則ロット配列（`lo[15]`）が直書きされている。
- **FATAL-B**: 配列オーバーラン（クラッシュ誘発）。14段目に達すると未定義の15番目の配列（`lo[15]`）を呼び出し、システムエラー（未定義動作）を引き起こす致命的バグ。
- **FATAL-C**: 发注方向とコメントの完全な乖離。BUY発注に"sell"、SELL発注に"BUY"とハードコーディングされており、コードの保守・検証が物理的に不可能。
- **D(補足)**: 永遠に実行されないデッドコードの存在。ループ内の無意味な `break` の直下に記述された追加発注ルートが物理的に遮断されている。
- **E**: スプレッド摩擦への無防備。加重平均+2.0pipsでの微益逃げ設定（`tp_2=2`）のため、スプレッド拡大時は利確前に実質マイナス域に叩き落とされる。

## CAP判定

- 未来リーク: 白(判断ロジック自体が不在のため該当せず)
- Close-cap: 黒(両建てロックにより含み損を抱えさせ、バックテスト上の最大DDを意図的に隠蔽・縮小する典型手法)
- プラセボ: 白(飾りとなるインジケーターパラメータは存在しない)
- 表示と実装の不一致: 黒(発注関数におけるBuy/Sellの方向とコメントの完全なる逆転)
- カーブフィット: 黒(ロット倍率配列 `{1,1,2,3,6...185}` が、特定の過去データで破綻しないよう手動で微調整された過剰最適化の典型)

## 数理

初期ロット0.1の場合、15段（150pips逆行）までナンピンが進むと累積ロットは803倍（80.3ロット）に達し、最終段の185倍ロット（18.5ロット）が直撃する。EURUSD等で年数回発生する150pipsのトレンド相場に遭遇した瞬間、数千万円規模のマイナス期待値を抱えて確実な口座全損（破綻）を迎える。
また、全決済の目標値が加重平均+2.0pipsであるため、Semura Lab想定の平均スプレッド0.6pips×全保有ポジション数の摩擦コストを差し引くと、実質的な利益はほぼ相殺される。圧倒的な破綻リスクに対し、見返りが無に等しく、数学的期待値は決定的にマイナスへ収束する。
