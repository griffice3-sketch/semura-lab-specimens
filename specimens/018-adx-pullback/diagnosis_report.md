
# 診断レポート／検体018

**診断：Semura Lab**

**検体番号：018**

**検体名：ADX Trend Pullback EA（DVN CORE）**

**分類：トレンドフォロー型・押し目エントリー系**

---

## 0. ロジックの目的・狙い・思想（解剖前の所見）

コードから逆算した設計思想は以下の通り。

- **思想**：「強いトレンドの中で、価格が移動平均線へ回帰した瞬間を捕らえる」古典的トレンドフォロー押し目戦略。
- **フィルター構造**：
  - ADX(14) > 25 かつ ADX上昇中（`adx1 > adx2`）＝トレンドが「存在し、かつ強化中」である局面のみ許可。
  - EMA20からの乖離をATRで正規化し、「前々足で離れていた（dist2≥0.5）→前足で接近した（dist1<0.5）」という**EMA回帰イベント**を検知。
  - 方向は+DI／-DIの大小で決定。
- **リスク設計**：SL＝ATR×1.5、TP＝SL×2.0（RR 2:1）。ATR連動で相場のボラティリティに適応。
- **執行**：新バー確定時のみ判定、単一ポジション限定、証拠金・約定モード・セッション確認を実装。

**総評コメント**：思想そのものは正統。素性は悪くない。だが「思想」と「実装」と「Semura規格」の間に致命的な断絶が複数存在する。以下、冷徹に解剖する。

---

## 1. 総合所見（生存能力の有無とその理由）

**生存能力：条件付きで「潜在的にあり」。ただし現状のままでは市場に出せば失血死する。**

理由：
- **誠実性は高い**。未来参照（先読み）は存在せず、確定足[1][2]のみを参照。約定は実際のBid/Ask、証拠金チェック、フィリングモード取得と、リアル口座を想定した実装で、**詐欺的病巣（CAP・未来関数・約定の魔法）は検出されず**。この点は評価する。
- **しかし「摩擦（スプレッド）への防御機構が完全欠落」しており、第1軸（生存と摩擦）で即死判定**。0.6pipsの摩擦を考慮するどころか、スプレッド拡大帯（ロールオーバー・指標時・薄商い時間）への一切の防御が無い。
- **バックテストデータが一切添付されていない**。PF・DD・勝率・取引回数——生存判定に必要な生体データがゼロ。よってロジックの「論理的堅牢性」は評価可能だが、「数値的優位性」は**検証不能**。

## 3. 発見された致命的な脆弱性

### 【致命傷①】スプレッド・時間帯フィルターの完全欠落（第1軸即死）
最大の病巣。トレンドフォロー×RR2:1は、そもそも**摩擦に対して脆弱**な構造（勝率が低めに出やすく、1トレードあたりの摩擦負担が相対的に重い）。にもかかわらず、スプレッド上限チェックも取引時間帯制限もない。**バックテストの固定スプレッドで生き延びても、可変スプレッドのリアル口座で確実に劣化する**典型パターン。

### 【致命傷②】ポジションサイジング不在＝月利目標の構造的達成不能
固定0.01ロット。SL距離はATRで毎回変動するのに、**リスク額（損失金額）が一定でない**——ATRが大きい局面では1トレードの損失額が膨らみ、小さい局面では縮む。「1トレードあたりリスク％固定」という規律が存在せず、月利10%は設計目標ではなく結果論。

### 【重大欠陥③】コメントと実装の論理矛盾
```
// Pullback event: just touched EMA on bar[2], now away on bar[1]
bool pullbackEvent = (dist1 < InpPullbackATRRatio) && (dist2 >= InpPullbackATRRatio);
```
コメントは「bar[2]で接触→bar[1]で離れた」と記述するが、**実装は逆**（bar[2]で離れていた→bar[1]で接近した）。つまり実装は「EMAへ**回帰中**の瞬間」を捕らえている。これは押し目の**天底ではなく途中**を掴む挙動であり、短期モメンタムに逆行して入る危険がある。**設計者自身が自分のロジックの挙動を誤認している可能性**を示す危険信号。承認前に意図の再確認が必須。

### 【欠陥④】エントリー方向とフィルターの整合性未検証
ADXは「上昇中」を要求するが、方向は+DI/-DIの瞬間値で決定。**トレンド方向とDI方向の一致確認がない**。理論上、ADX上昇＝トレンド強化だが、押し目回帰の瞬間にDIが一時反転していると、トレンドに逆張りする方向でエントリーする穴が残る。

### 【欠陥⑤】検証データの不在
再現性・堅牢性を語る土台となる数値が一切ない。**これ自体がSemura規格では重大な手続き違反**。

## 4. 処方箋・改善指示（Semura Lab規格への再生手順）

**優先度A（生存必須）**
1. **スプレッドフィルター実装**：`SymbolInfoInteger(_Symbol, SYMBOL_SPREAD)` を取得し、設計摩擦0.6pips＋αを上限とする。上限超過時はエントリー全面回避。
2. **時間帯フィルター実装**：ロールオーバー前後（例：サーバー時間23:00〜01:00）および主要指標時間帯を除外。バックテストも可変スプレッド／ティックデータで再実行すること。
3. **リスク基準ロット化**：固定0.01を廃止。「口座残高×リスク％ ÷ SL距離（ATR×1.5）」でロットを算出し、1トレードあたりリスクを一定化。これで初めて「月利」という率目標が意味を持つ。

**優先度B（論理の誠実性）**
4. **コメントと実装の矛盾を解消**：pullbackEventの意図（回帰途中か、反発後か）を明文化し、コードと一致させよ。意図が「反発後の順張り」なら条件を反転させ、`close1`とEMAの位置関係（トレンド方向側にあるか）も追加せよ。
5. **方向整合フィルター追加**：BUYなら`close1 > ema1`かつ`+DI>-DI`、SELLなら逆、を必須条件化し、トレンドと逆行するエントリーを排除。

**優先度C（承認手続き）**
6. **13年ウォークフォワード分析結果を提出**：IS/OOS分割、可変スプレッド、実効スワップ込み。PF・最大DD・取引回数・OOS劣化率を明示。数値なき承認はありえない。

## 総合評価

# 判定：**差し戻し（Re-submission Required）**

**論拠**：詐欺的病巣・未来参照・過剰最適の痕跡は認められず、**素性（誠実性）は合格ライン**。ナンピン・マーチン厳禁、単一ポジションも遵守。この点で「不採用（廃棄）」には値しない。

しかし——
- スプレッド耐性の完全欠落（第1軸即死）
- ポジションサイジング不在による月利目標の構造的達成不能
- コメントと実装の論理矛盾（設計意図の疑義）
- 検証データの全面欠落

以上により、**現状のまま採用は断固不可**。ただし骨格は健全であり、上記処方箋A〜Cの完遂と再検証データの提出をもって**再審査に値する（再構築ではなく差し戻しで足る）**と判断する。

**次回提出時、優先度Aの3項目とバックテスト実データが揃わぬ限り、再診は行わない。**
