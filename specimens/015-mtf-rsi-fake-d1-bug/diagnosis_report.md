
# 診断カルテ — 検体015

**Semura Lab／トレードロジック診断部**
**検体番号：015（旧称 "VR Rsi Robot" 系統）**

## 0. ロジックの目的・狙い・思想の解剖

提出者の設計意図を逆算する。

- **思想**：上位足（D1）と下位足（H1）の2つのRSIを「合議制」とし、両者がオーバーソールド帯（20）から上向き反転、またはオーバーボート帯（80）から下向き反転した瞬間に順張りエントリーする。
- **狙い**：上位足でトレンドの大局を、下位足でタイミングを取る——いわゆる「環境認識フィルター」型のモメンタム追従。
- **ポジション設計**：買い1・売り1の最大2ポジション。逆シグナルで反対ポジションをドテン決済する単純なロジック。

思想自体は教科書的で悪くない。**問題は、その思想がコード上で一切実装されていない**点にある。以下、冷徹に解剖する。

---

## 1. 総合所見（生存能力の有無）

**生存能力：なし（臨床的死亡）。**

本検体は「マルチタイムフレーム合議制」を標榜しながら、診断対象部のコードを見る限り**D1の参照が完全に欠落し、H1の同一値を2重に評価しているだけ**である。つまり設計思想と実装が乖離した「死産個体」。

さらに致命的なのは、**この検体には損切り（SL）も利確（TP）も存在しない**こと。`OrderSend` のSL/TPはともに `0`。ポジションはドテンシグナルが来るまで永久に塩漬けされる。これは「優位性の検証」以前の、リスク管理の完全な不在を意味する。

生存基盤がないため、月利・PF以前の段階で**棄却**となる。

## 3. 発見された致命的脆弱性

### 病巣①：D1フィルターの実装欠落（設計詐称）
```
rsi_mass_d0 = iRSI(_Symbol, iRSI_TimeFrame_H1, iRSI_Period_H1, ...); // D1のはずがH1
```
`iRSI_TimeFrame_D1` も `iRSI_Period_D1` も**一度も使われていない**。コンパイラから見れば、D1条件は単なるH1条件の重複であり、合議制は最初から1票制。「マルチタイムフレーム」は看板倒れ。**プラセボ・パラメータ**である。

### 病巣②：損切り・利確の完全不在（最大の死因）
SL=0, TP=0。決済はドテンシグナル待ちのみ。トレンドが一方向に伸び続ければ、含み損ポジションは無限に拡大し、`CheckMargin` が通らなくなるまで放置される。**これは戦略ではなく自殺装置。**

### 病巣③：摩擦・時間帯防御ゼロ
スプレッド拡大帯（早朝・指標発表時）の回避ロジックが皆無。0.6pipsどころか、拡大時の数pips摩擦をそのまま被弾する設計。

### 病巣④：APIの世代混在（技術的不整合）
ファイル名は `.mq5` だが、中身は `OrderSend(旧シグネチャ)`／`OrderSelect(SELECT_BY_POS)`／`MarketInfo`／`Ask`/`Bid` 等、**MQL4の構文**。MT5環境では原則コンパイル不能。**そもそも「動く」という前提が虚構**であり、提出されたバックテスト結果が存在するなら、それ自体が出所不明の疑義対象となる。

### 病巣⑤：0.01ロット固定
資金管理ロジック不在。複利設計もリスク率設計もない。月利10%という目標と構造的に矛盾。

## 4. 処方箋（Semura Lab規格への再生指示）

再生は可能だが、**ほぼ全臓器の移植**を要する。以下を厳守せよ。

1. **環境の確定**：MQL4で運用するならファイルとビルド環境をMT4に統一。MT5なら`CTrade`クラス・`CopyBuffer`へ全面書き換え。**まず「動く個体」にせよ。**

2. **D1フィルターの蘇生**：
   ```
   rsi_mass_d0 = iRSI(_Symbol, iRSI_TimeFrame_D1, iRSI_Period_D1, PRICE_CLOSE, 1);
   rsi_mass_d1 = iRSI(_Symbol, iRSI_TimeFrame_D1, iRSI_Period_D1, PRICE_CLOSE, 2);
   ```
   標榜した思想を実装で裏付けよ。

3. **SL/TPの強制実装**：ATRベースの動的損切り（例：1.5×ATR）と、最低でもリスクリワード1:1.2以上の利確を必須化。**放置ポジションは1つも許容しない。**

4. **スプレッドフィルター**：エントリー直前に `Ask-Bid` を取得し、0.6pips（または銘柄別閾値）超過時はエントリー棄却。加えてGMTベースの時間帯フィルターで早朝・主要指標時を除外。

5. **資金管理の導入**：固定ロットを廃し、口座残高×リスク率（例1〜2%）÷SL距離でロット算出。月利目標と整合させよ。

6. **新規バー判定**：現状ティック毎評価のため、同一バー内で条件が点滅する。`Time[0]` 監視で1バー1判定に限定し、過剰取引を抑制せよ。

## 総合評価

# 【不採用 — 要再構築（差し戻し以前）】

検体015は、診断台に乗せる以前の問題を抱えている。
- 設計思想（MTF合議制）は実装されておらず**詐称状態**。
- 損切りが無く、**リアル口座では遅かれ早かれロスカットで死亡する**自然摂理違反個体。
- そもそもMT4/MT5の構文が混在し、**「動作している」という前提自体が疑わしい**。

唯一の美点は「未来参照が無いこと」と「多重保有をしないこと」のみ。だがそれは、生きていることの証明にはならない。

**処方箋に従い、SL/TP・フィルター・D1蘇生・資金管理を移植して初めて「再構築可」のテーブルに着く。** 現状のまま実弾投入することは、Semura Labとして厳に禁ずる。
