# 構造診断レポート — 検体002

**診断者: Semura Lab.**
**対象: トレンドフォロー型スキャルピングEA(MT4 / 売買執行型)**

---

## 序文 — このロジックは何か

本検体は、MT4向けの自動売買プログラム(EA)である。ファイルに付随する説明文は確認できなかったため、本診断では構造の事実のみから設計意図を逆算して記述する。

コードに残る痕跡から、当初の設計意図は次のものであったと読み取れる。上位足(H1)のパラボリックSARで相場の方向を判定し、ロンドン〜ニューヨーク時間帯(既定で 02:30〜21:00、サーバー時刻)に限定して、狭い値幅を狙う短期売買(スキャルピング)を行う——という構想である。SAR・買いシグナル・売りシグナルを組み立てるコード片が、コメントとして本体に残されている。

しかし実装は、その構想に達していない。方向を判定するロジックは全てコメントアウトされ、発注方向は「売り」を表す定数に固定されている。すなわち稼働時、本検体は相場が上昇局面か下降局面かを一切判定せず、ひたすら売りの待機注文(SELLSTOP)を出し続ける。トレンドフォローでも、押し目買い・戻り売りでもない。方向選択という概念そのものが、稼働コードからは失われている。

この構造は、自動売買が「勝てない」と評される典型的な理由をいくつも内包する。第一に、方向の優位性(エッジ)が存在しないため、エントリーの良し悪しが市場のランダムな動きに委ねられる。第二に、決済幅と損切り幅の建て付けに、約定前からスプレッドが静かに上乗せされる。第三に、ロットサイズが「リスク率」と表示されながら、実際には損切り幅も1pipの金額価値も参照していない。バックテストの数字と実口座の挙動が乖離する、コツコツ積んでドカンと失う——こうした帰結は、いずれも本検体の構造から数理的に予測できる。以下、物証(該当コード行)と数理のみで示す。

---

## 第1章 仕様照合 — 表示と実装の対応

入力パラメータの表示名と、実装での扱いを照合する。

- **発注方向**: 入力欄に方向選択の項目は存在しない。実装(`OpenOrder()`)では `OrdType` に売りを表す定数 `OP_SELL` が直接代入され、以降変更されない。方向を切り替える分岐(買いシグナル・売りシグナル)は全てコメントアウトされている。**表示上は中立的な「スキャルピングEA」を装うが、稼働実態は売り専用である。**
- **`InpRisk`(Risk Percentage of Balance)**: 「残高に対するリスク率(%)」と表示。実装(`CalculateVolume()`)では余剰証拠金に係数を掛けて10万で除すのみで、損切り幅もpip価値も参照しない。**「リスク率」という表示と、実装の「証拠金比例係数」が一致しない。**
- **`InpSL_Pips`(3.5) / `InpTP_Pips`(7)**: 損切り・利確の値幅として表示。ただし実装では損切り・利確とも、エントリー価格の算出に `InpSL_Pips/2`(=1.75)を用いており、表示値そのままの建て付けにはならない(第3章)。
- **`InpProfitStep`(Trailing Step) / `InpPeriodKelner`**: 入力候補としてソース内に存在するが、いずれもコメントアウトまたは未接続で、計算に関与しない(第4章 飾りパラメータ)。

---

## 第2章 致命バグ — 稼働を不全にする構造

### FATAL-A: 方向判定ロジックの不在(売り固定)

最重要の事実。発注方向は定数 `OP_SELL` に固定されている。本来方向を決めるはずのパラボリックSAR、および買い/売りシグナルによる分岐は、すべてコメントアウトされ稼働しない。

結果として本検体は、上昇相場・下降相場・レンジを区別せず、常に売りの待機注文のみを発注する。方向選択の優位性はゼロであり、エントリーの当否は市場のランダムな変動に委ねられる。これは戦略の不全ではなく、戦略の不在である。

### FATAL-B: 実効損切り幅へのスプレッド上乗せ

売り待機注文の建て付けは、エントリー価格を `Bid - 1.75pips`、損切りを `Ask + 1.75pips` に置く。両者の差(実効的な損切り幅)は次のようになる。

```
実効SL距離 = (Ask + 1.75) − (Bid − 1.75) = (Ask − Bid) + 3.5 = スプレッド + 3.5pips
```

名目の損切り 3.5pips に対し、約定が成立する前からスプレッド分が静かに上乗せされる。利確側にこの上乗せはない。**損切りだけが市場コストぶん深くなる非対称が、設計の出発点に組み込まれている。**

### FATAL-C: ロットサイズが損切り幅を参照しない

`CalculateVolume()` は `InpRisk × 余剰証拠金 ÷ 100000` を基にロットを算出する。損切り幅(pips)も、1pipあたりの金額価値も、計算式に登場しない。これは「資金の何%を損切りで失うか」を制御するリスク率ではなく、単なる証拠金比例の固定係数ロットである。残高が増えればロットは制限まで膨張し、1トレードの金額リスクは設計者の想定から外れて拡大する。

## 第3章 CAP監査 — 事実の歪曲に関する判定

Semura Lab. はCAP(事実の歪曲)の有無を、以下の5観点で判定する。判定は 黒(該当)/ 灰(疑いあり・断定不可)/ 白(該当なし) で示す。

- **未来リーク(未確定足の参照): 白**
  本検体には市場状況を参照して売買を判断するロジックthat自体が存在しない(売り固定)。参照すべき指標値がないため、未来リークは構造的に発生しない。「健全だから白」ではなく「判断ロジックが無いため該当しない」という意味の白である。

- **Close-cap(損益のSL値での頭打ち): 白**
  バックテストの損益を人為的に頭打ちにする操作は見られない。

- **プラセボ(機能するように見えて稼働しない飾り): 黒**
  本検体で最も濃い項目。方向判定の核であるSAR・買い/売りシグナルが全てコメントアウト(死にコード)。トレーリング刻み幅(`InpProfitStep`)、期間パラメータ(`InpPeriodKelner`)も未接続。入力欄やコード上は多機能を装うが、稼働するのは「売りを出し続ける」一点のみ。**設計の中核が飾りである。**

- **表示と実装の不一致: 黒**
  「Risk Percentage」と表示しながら損切り幅を参照しない(第2章 FATAL-C)。方向選択を装いながら売り固定(FATAL-A)。表示と稼働実態が複数箇所で乖離している。これは作者の意図を断罪するものではなく、コードに付随する表示と実装の間に客観的な乖離が存在するという技術事実である。

- **カーブフィット(過剰最適化): 灰**
  最適化すべきシグナルパラメータ自体が稼働していないため、過剰最適化の痕跡を確認できない。判断材料が存在しないため灰とする。

## 第4章 摩擦の数理 — 期待値による帰結

本検体が構造的に利益を生まないことを、数理のみで示す。トレード1回の期待値がゼロとなる勝率(損益分岐勝率 p)は次式で与えられる。

```
p = (実効SL + スプレッド) / (実効TP + 実効SL)
```

名目パラメータ(TP=7, SL=3.5)に第2章の実効建て付けを反映し、スプレッドを変えて算出する。

- スプレッド 0.0pips: 損益分岐勝率 ≈ 33.3%
- スプレッド 0.6pips: 損益分岐勝率 ≈ 42.3%
- スプレッド 1.0pips: 損益分岐勝率 ≈ 47.8%
- スプレッド 2.0pips: 損益分岐勝率 ≈ 60.0%

ここで本検体の特異性が際立つ。名目上はTP(7) > SL(3.5)の「利大損小」であり、損益分岐勝率は一見低い(コストゼロなら33%で足りる)。**それでも構造的に負ける。理由は勝率がこの分岐点を超える根拠がどこにも存在しないことにある。**

方向判定が無効化され売りに固定されている以上(FATAL-A)、ランダムに変動する相場における方向的中の期待値は50%に過ぎない。一方、現実的なスプレッド0.6〜1.0pipsを織り込んだ損益分岐勝率は42〜48%——これは「コストを賄うだけ」のラインであり、利益を生むにはそれを継続的に上回り続けねばならない。方向のエッジを持たない売り固定の戦略が、それを達成する数理的根拠はない。

加えて、待機注文に対するトレーリング(第5章 補足)が約定タイミングと建て付けを変動させるため、実効TP/SLは設計値からさらに揺らぐ。揺らぎは期待値の改善には寄与しない。

結論として、本検体の1トレードあたり期待値は、スプレッドと実効建て付けの非対称ぶんだけ構造的に負へ傾く。試行回数を重ねるほど、収支は数理的予測通りマイナスへ収束する。

## 第5章 規格適合 — Semura Lab. 基準との照合

Semura Lab. が健全な売買ロジックに求める基準と照合する。

- **リスクリワード 1:1(TP/SL 5〜8pips)**: 不適合。名目はTP7/SL3.5だが、方向エッジが無いため比率の良否が収支に結びつかない。
- **絶対摩擦(スプレッド管理・高スプレッド時の回避)**: 部分的のみ。スプレッド上限チェックは存在するが、桁補正が効かず、かつ実効損切りへのスプレッド上乗せ(FATAL-B)は回避できていない。
- **ポジション管理(常に1ポジのみ)**: 待機注文を含むポジション状態の管理は試みられているが、トレーリングが未約定注文の建値を追従させるため、ポジションの定義自体が不安定。
- **目標(勝率55% / PF 1.2〜1.5 / 月利10%)**: 到達不能。方向エッジ不在(FATAL-A)により、勝率55%を継続達成する数理的根拠が存在しない。

**規格適合: 不適合。** 本検体は、売買戦略として機能するために必要な「方向の優位性」を稼働コードに持たない。表示される多機能は大半が飾り(プラセボ)であり、実効建て付けはスプレッドぶん負へ傾く。試行を重ねるほど収支は数理的にマイナスへ収束する構造である。

---

### 補足 — トレーリング構造について

本検体のトレーリング処理は、未約定の待機注文(SELLSTOP/BUYSTOP)に対しても建値・損切り・利確を毎ティック再設定し、現在価格に追従させる。待機注文が価格を追い続けるため、「どの価格で待ち伏せていたか」という指値の意味が曖昧になる。約定済みポジションのトレーリングとは性質が異なり、エントリー価格の事後的な変動要因となる。これ自体は損益を直接歪曲する操作ではないが、バックテストと実口座の挙動が乖離する一因となりうる。
