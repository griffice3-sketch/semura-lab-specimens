# 検体002 — 検体カード

| 項目 | 内容 |
|------|------|
| 検体番号 | 002 |
| 型 | トレンドフォロー型スキャルピングEA(MT4 / 売買執行型) |
| 想定設計意図 | H1パラボリックSARで方向判定 → ロンドン〜NY時間帯にスキャルピング(コードから逆算) |
| 稼働実態 | 方向判定が無効化され、売り(SELLSTOP)に固定 |
| 総行数 | 589行 |
| 診断者 | Semura Lab. |

## 診断サマリ

- **FATAL-A**: 方向判定ロジック(SAR・買い/売りシグナル)が全てコメントアウト → `OP_SELL` 固定。方向の優位性が存在しない。
- **FATAL-B**: 売り建て付けで実効損切り幅にスプレッドが上乗せ(`(Ask−Bid)+3.5pips`)。利確側は非対称に上乗せなし。
- **FATAL-C**: ロット計算が損切り幅・pip価値を参照しない。「リスク率」表示だが実体は証拠金比例の固定係数。
- **D(補足)**: トレーリングが未約定の待機注文の建値を現在価格に追従させる。
- **E**: 飾りパラメータ(`InpProfitStep`・`InpPeriodKelner` 等)が未接続。

## CAP判定

- 未来リーク: 白(判断ロジック自体が不在のため該当せず)
- Close-cap: 白
- プラセボ: 黒(設計中核が死にコード)
- 表示と実装の不一致: 黒
- カーブフィット: 灰(最適化対象が稼働していないため判定不能)

## 数理

名目はTP7/SL3.5の利大損小で損益分岐勝率は低い(コストゼロで約33%)。しかし方向エッジ不在(売り固定)のため、勝率がこの分岐点を超える根拠が存在せず、期待値は構造的に負。検体001(利小損大でRR反転)とは異なり、「RRは良好だがエッジ皆無で負ける」パターン。
