# 記事公開・コード埋め込み手順（Zenn / Medium 共通運用）

このアーカイブの検体コードを、ZennとMediumの記事へ埋め込むための手順。
**正本はこのGitHubリポジトリ、記事への埋め込みは用途に応じてGist or リンク**という二層構成で運用する。

---

## 基本方針

| 置き場 | 役割 |
|--------|------|
| GitHubリポジトリ（本体） | 検体の正本。カタログ・診断カードを集約。永続リンク先 |
| GitHub Gist | 記事に「埋め込む」ための写し。Zenn/Medium両対応 |
| 記事本文 | 問題該当箇所の**抜粋コードブロック**＋全文への導線 |

理由：ZennもMediumも記事へのファイル添付機能が無い。両者に共通して使えるのが
GitHub Gistの埋め込み。リポジトリのファイル内容をそのままGistにも置けば、
1つの素材で両プラットフォームに対応できる。

---

## 手順A：Gistを用意する（両プラットフォーム共通・初回のみ）

1. https://gist.github.com を開く
2. ファイル名を付ける（例：`specimen001_excerpt.mq4`）— 拡張子を付けると構文ハイライトされる
3. `specimens/001-.../specimen_excerpt.mq4` の中身を貼り付け
4. **必ず「Create public gist」を選ぶ**（secret gist は Medium に埋め込めない）
5. 作成後のGist URLを控える

> 補足：Mediumは単一ファイルだけの埋め込みに非対応で、Gist内の全ファイルが表示される。
> よって**1検体＝1 Gist（1ファイル）**で作るのが最も綺麗。

---

## 手順B：Zennへの掲載

Zennは2つの手段を併用できる。

### B-1. Gist埋め込み（行番号付きで全抜粋を見せる）
記事本文に次の記法を置く：
```
@[gist](https://gist.github.com/＜ユーザー名＞/＜GistのID＞)
```

### B-2. 折りたたみ＋抜粋コードブロック（本文内で完結させたい場合）
Zennは `<details>` に対応する：
```
<details><summary>該当コードを見る（決済不発バグ）</summary>

​```cpp
if( order.IsOpen()
    && ((atr < TRH) && (profit >= TP_LV))
    && ((atr > TRH) && (profit >= TP)) )   // ← AND は恒偽
{ order.Close(); }   // 到達しない
​```

</details>
```

推奨：**B-1で全抜粋をGist埋め込み + 本文の各診断項目にB-2で該当数行**。

---

## 手順C：Mediumへの掲載

Mediumは `<details>` 折りたたみ非対応（生HTMLを通さない）。Gist埋め込みで対応する。

### C-1. Gist埋め込み
1. 記事編集画面で `+` →「Add an embed」を選ぶ
2. 手順AのGist URLを貼る
3. **Enter/Returnキーを必ず押す**（押さないとURLがテキストのまま残る）
4. Gistが行番号・ハイライト付きで記事内に表示される

### C-2. 本文の抜粋
Mediumの素のコードブロック（Command/Ctrl+Alt+6、またはバッククォート3つ）で
各診断項目の該当数行を貼る。構文の色付けは弱いが、要点を示すには十分。

推奨：**各診断項目はC-2の短い抜粋で見せ、全抜粋はC-1のGist埋め込みに集約**。

---

## 手順D：公開後の後始末

1. 記事URLを、該当検体の `NOTE.md` の「診断レポート本体」欄に記入
2. リポジトリ `README.md` のカタログ表「診断記事」列にリンクを記入
3. コミット＆プッシュ

---

## 新検体の追加手順（002以降）

1. `_TEMPLATE/` を `specimens/00X-＜検体名＞/` として複製
2. `specimen_excerpt.<ext>` に該当箇所の構造抜粋を入れる
3. `NOTE.md` を埋める
4. `README.md` のカタログ表に1行追加
5. 手順A〜Dで記事公開
